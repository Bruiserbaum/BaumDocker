# Mailcow

A full self-hosted email server suite. Includes Postfix (SMTP), Dovecot (IMAP/POP3), Rspamd (spam filtering + DKIM), ClamAV (antivirus), SOGo (webmail + CalDAV/CardDAV), automatic Let's Encrypt SSL, and a web-based admin UI — all managed as a single stack.

> **Note:** Mailcow is architecturally different from the other stacks in this repo. It consists of 17 tightly versioned services and uses a generated `mailcow.conf` instead of a plain `.env`. The official [`mailcow-dockerized`](https://github.com/mailcow/mailcow-dockerized) repo is the authoritative source for the compose file — recreating it manually risks version mismatches that break mail delivery. This README covers setup via their repo and via Portainer.

## Services

| Service | Port(s) | Description |
|---------|---------|-------------|
| **Nginx** | 80, 443 | Web UI, webmail, and reverse proxy |
| **SOGo** | — (internal) | Webmail, CalDAV, CardDAV |
| **Postfix** | 25, 465, 587 | SMTP + SMTP submission |
| **Dovecot** | 143, 993, 110, 995, 4190 | IMAP, IMAPS, POP3, POP3S, Sieve |
| **Rspamd** | — (internal) | Spam filtering and DKIM signing |
| **ClamAV** | — (internal) | Antivirus scanning |
| **MariaDB** | — (internal) | Database |
| **Redis** | — (internal) | Cache and session store |
| **Unbound** | — (internal) | Internal DNS resolver |
| **Acme** | — (internal) | Automatic Let's Encrypt cert management |

## Prerequisites

Before deploying, ensure:

1. **A domain you control** — you'll need to set DNS records (MX, SPF, DKIM, DMARC)
2. **A public static IP** with port 25 not blocked by your ISP (many residential ISPs block this — check before proceeding)
3. **Reverse DNS (PTR) record** set by your ISP or hosting provider to match `MAILCOW_HOSTNAME`
4. **Ports open on your firewall:** 25, 80, 443, 465, 587, 143, 993, 110, 995, 4190

## DNS Records

Set these on your domain before starting:

| Type | Name | Value |
|------|------|-------|
| A | `mail` | Your server's public IP |
| MX | `@` | `mail.yourdomain.com` (priority 10) |
| TXT | `@` | `v=spf1 mx ~all` |
| TXT | `_dmarc` | `v=DMARC1; p=quarantine; rua=mailto:dmarc@yourdomain.com` |
| CNAME | `autoconfig` | `mail.yourdomain.com` |
| CNAME | `autodiscover` | `mail.yourdomain.com` |

> DKIM records are generated by Mailcow after first start — copy them from **Mail Setup → Configuration → ARC/DKIM keys** in the admin UI.

## Setup (CLI)

### 1. Clone the official Mailcow repo

```bash
git clone https://github.com/mailcow/mailcow-dockerized /opt/mailcow
cd /opt/mailcow
```

### 2. Generate the config

```bash
./generate_config.sh
```

Enter your hostname (e.g. `mail.yourdomain.com`) and timezone when prompted. This creates `mailcow.conf`.

### 3. Review and edit mailcow.conf

See `mailcow.conf.example` in this folder for a documented reference of all key settings, including how to disable ClamAV or Solr to reduce RAM usage.

### 4. Start the stack

```bash
docker compose pull
docker compose up -d
```

First start takes several minutes as images are pulled and the DB is initialized.

### 5. Access the admin UI

Navigate to `https://mail.yourdomain.com`

Default credentials (change immediately):
- **Username:** `admin`
- **Password:** `moohoo`

### 6. Add your DKIM records

Go to **Mail Setup → Configuration → ARC/DKIM keys**, generate keys for your domain, and add the resulting TXT record to your DNS.

## Setup (Portainer)

Mailcow's compose file is maintained in the official repo and must be deployed from there — do **not** point Portainer at this BaumDocker repo for Mailcow.

1. Go to **Stacks → Add stack → Repository**
2. Fill in:

| Field | Value |
|-------|-------|
| Repository URL | `https://github.com/mailcow/mailcow-dockerized` |
| Repository reference | `refs/heads/master` |
| Compose path | `docker-compose.yml` |

3. Under **Environment variables**, add every value from `mailcow.conf.example` in this folder
4. Click **Deploy the stack**

> **Large image warning:** Mailcow pulls 17 images on first deploy and will likely hit Portainer's 504 timeout. Pre-pull on the host before deploying:
> ```bash
> cd /opt/mailcow && docker compose pull
> ```
> Then deploy from Portainer — it will use the cached images.

## RAM Requirements

| Configuration | Minimum RAM |
|---------------|-------------|
| Full stack (ClamAV + Solr enabled) | 6 GB |
| ClamAV disabled (`SKIP_CLAMD=y`) | 3 GB |
| ClamAV + Solr disabled | 2 GB |

Set `SKIP_CLAMD=y` and `SKIP_SOLR=y` in `mailcow.conf` for low-memory hosts.

## Behind a Reverse Proxy (e.g. Nginx Proxy Manager)

If Nginx Proxy Manager is already on ports 80/443, change Mailcow's HTTP/HTTPS ports in `mailcow.conf`:

```
HTTP_PORT=8080
HTTPS_PORT=8443
HTTP_BIND=127.0.0.1
HTTPS_BIND=127.0.0.1
```

Then set `SKIP_LETS_ENCRYPT=y` and let NPM handle SSL. In NPM, proxy `mail.yourdomain.com` to `127.0.0.1:8080` and enable SSL there.

> Mail ports (25, 465, 587, 143, 993, etc.) must still be exposed directly — they cannot go through a standard reverse proxy.

## Updating

```bash
cd /opt/mailcow
git pull
docker compose pull
docker compose up -d
```

Check the [Mailcow changelog](https://github.com/mailcow/mailcow-dockerized/releases) before updating — some releases require manual migration steps.

## Backup

Mailcow includes a built-in backup script:

```bash
cd /opt/mailcow
./helper-scripts/backup_and_restore.sh backup all
```

Backups are written to `/opt/mailcow/backup` by default. Schedule this with cron for automated backups.

To restore:

```bash
./helper-scripts/backup_and_restore.sh restore
```
